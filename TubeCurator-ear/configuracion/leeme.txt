<h2>Notas</h2>
<h3>Recurso duplicado al deployar</h3>
Al deployar puede aparecer la leyenda "Duplicate resource". Esto se debe a que el deployer de las jbosstools del Eclipse agrega un deployer scanner al standalone.xml haciendo que se deployen las cosas dos veces. Para arreglarlo:
<ul>
<li> En la configuracion del server (doble click sobre el nombre del server en la pesta√±a 'Servers' del Eclipse), buscar la seccion Deployment Scanners y deschequear 'Add missing deployment scanners...' y 'Remove addeded deployemnt scanners...' </li>
<li> En el standalone.xml buscar dentro del subsystem deployment-scanner y remover la seccion que agrego las jbosstools: </li>
```
        <subsystem xmlns="urn:jboss:domain:deployment-scanner:1.1">
            <deployment-scanner path="deployments" relative-to="jboss.server.base.dir" scan-interval="5000"/>
            
            REMOVER ESTA LINEA!!
            <deployment-scanner name="jbosstoolsscanner1" path="E:\Temp\jboss711Pruebas\standalone\deployments" scan-interval="5000" deployment-timeout="60"/>
                        
        </subsystem>

```
<li> Reiniciar el server </li>
</ul>
